Quero que você analise esta seção do protótipo no Figma e gere o código correspondente com alta fidelidade visual e estrutural.

Objetivo:
converter este trecho do design em uma implementação frontend pronta para uso no projeto, utilizando HTML, CSS e JavaScript.

Instruções:

Reproduza a interface com o máximo de precisão possível.

Mantenha a responsividade em diferentes tamanhos de tela.

Estruture o código de forma organizada, reutilizável e fácil de manter.

Use HTML semântico.

Use CSS bem estruturado, com nomes de classes claros e consistentes.

Use JavaScript apenas quando necessário para interações da interface.

Preserve tipografia, cores, ícones, espaçamentos, sombras, bordas, tamanhos e alinhamentos.

Não simplifique demais o layout.

Não adicione elementos, textos ou componentes que não existam no protótipo.

Se algum detalhe estiver ambíguo, siga a solução visual mais coerente com o restante da interface.

Pense no padrão de código como se essa tela fosse continuar sendo expandida por outros desenvolvedores.

Evite redundância e repetições desnecessárias no HTML, CSS e JavaScript.

Caso existam elementos repetidos, organize a estrutura para facilitar reutilização futura.

Considere estados visuais como hover, foco, ativo e desabilitado quando fizer sentido.

Gere um resultado fiel ao design e preparado para manutenção futura.

Tecnologia obrigatória:

HTML

CSS

JavaScript

Entrega esperada:

arquivo HTML completo

arquivo CSS separado

arquivo JavaScript separado, se necessário

código pronto para integração no projeto

estrutura clara e organizada

Se houver componentes como:

botões

cards

inputs

tabelas

menus

modais

abas

listas
mantenha consistência visual e estrutural entre eles.

Prioridades:

Fidelidade visual ao protótipo

Organização do código

Responsividade

Facilidade de manutenção

Clareza na estrutura
